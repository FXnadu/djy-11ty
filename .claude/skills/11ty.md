# Eleventy (11ty) 项目规范指南

> 参考来源：[11ty.dev/docs](https://www.11ty.dev/docs/) 官方文档与最佳实践

---

## 1. 配置文件

### 文件命名
- **Eleventy v3+**：优先使用 `eleventy.config.js`（ESM 语法）
- **旧版兼容**：`.eleventy.js`（CommonJS）
- 项目中同时存在时，`eleventy.config.js` 优先级更高

### 配置结构
```js
// eleventy.config.js (ESM, 推荐)
export default function(eleventyConfig) {
  // 插件、过滤器、集合、短代码...
  return {
    dir: {
      input: "src",
      output: "_site",
      includes: "_includes",
      data: "_data"
    }
  };
};
```

### 模块化拆分
将大型配置拆分到 `eleventy/` 目录下的独立模块：
```
eleventy/
  config/
    collections.js    # 集合定义
    computed.js       # 计算字段
    filters.js        # 过滤器
    passthrough.js    # 静态资源复制
    content-types.js  # 内容类型配置
    post-utils.js     # 文章工具函数
    category-meta.js  # 分类元数据
  utils/
    path-utils.js     # 路径工具
    slug-encoder.js   # slug 编码
```

在主配置中引入：
```js
const { registerCollections } = require("./eleventy/config/collections");
const { registerDateFilters } = require("./eleventy/config/filters");
```

## 2. 目录结构（推荐）

```
src/
  _data/              # 全局数据文件（JSON / JS / 11tydata.js）
    siteConfig.js     # 站点配置
    build.json        # 构建信息
    categoryMeta.json # 分类元数据
  _includes/          # 布局与可复用组件
    layouts/
      base.njk        # 基础布局
      post.njk        # 文章布局
    partials/
      head.njk        # <head> 部分
      site-header.njk # 页头
      site-footer.njk # 页脚
  assets/
    css/              # 样式文件
    js/               # 脚本文件
  content/
    posts/            # 文章内容（Markdown）
    pages/            # 独立页面（Nunjucks）
    moments/          # 短内容/动态
  static/             # 不经处理的静态文件（robots.txt 等）
```

### 关键目录说明
- **`_includes/`**：存放布局和 partials，不会被当作独立页面输出
- **`_data/`**：全局数据，通过 `{{ siteConfig.xxx }}` 在模板中访问
- **`content/`**：实际内容文件，每个文件生成一个页面

## 3. 数据级联（Data Cascade）

数据按优先级从高到低合并：

1. **Front matter**（模板内，最高优先级）
2. **目录数据文件**（`posts.11tydata.js`）
3. **父目录数据文件**（逐级向上继承）
4. **全局数据文件**（`_data/` 目录）
5. **计算数据**（`eleventyComputed`）
6. **默认值**（最低优先级）

### 目录数据文件
在内容目录旁放置同名 `.11tydata.js` 文件，为该目录下所有模板设置默认值：
```js
// src/content/posts/posts.11tydata.js
module.exports = {
  layout: "layouts/post.njk",
  tags: ["post"],
  permalink: "/posts/{{ page.fileSlug }}/"
};
```

### 计算字段（Computed Data）
使用 `eleventyComputed` 动态生成数据：
```js
// eleventy/config/computed.js
module.exports = {
  getComputedFields() {
    return {
      permalink: (data) => {
        // 根据目录结构动态决定 URL
      },
      eleventyNavigation: (data) => {
        // 自动生成导航数据
      }
    };
  }
};
```

### 全局数据
- 通过 `eleventyConfig.addGlobalData()` 注册
- 或在 `_data/` 目录放置 `.json` / `.js` 文件
- 模板中通过变量名直接访问：`{{ build.timestamp }}`

## 4. 模板语言

### 推荐选择
- **Nunjucks (`.njk`)**：功能最全，支持继承、宏、异步，推荐用于布局和复杂页面
- **Markdown (`.md`)**：文章内容首选，支持 front matter
- **JavaScript (`.11ty.js`)**：需要编程逻辑时使用，可动态生成内容

### Nunjucks 布局继承
```njk
{# base.njk #}
<!DOCTYPE html>
<html>
<head>
  {% include "partials/head.njk" %}
</head>
<body>
  {% block content %}{% endblock %}
  {% include "partials/site-footer.njk" %}
</body>
</html>
```

```njk
{# post.njk #}
{% extends "layouts/base.njk" %}
{% block content %}
  <article>
    <h1>{{ title }}</h1>
    {{ content | safe }}
  </article>
{% endblock %}
```

### Front Matter
```yaml
---
title: 文章标题
description: 文章描述
date: 2025-03-10
tags: [post, tutorial]
category: 电商笔记
---
```

## 5. 集合（Collections）

### 基于标签的集合
在 front matter 中添加 `tags`，自动创建集合：
```yaml
---
tags: ["post", "tutorial"]
---
```

模板中访问：`{% for post in collections.post %}`

### 自定义集合
```js
// eleventy/config/collections.js
module.exports = {
  registerCollections(eleventyConfig) {
    eleventyConfig.addCollection("tutorials", (collectionApi) => {
      return collectionApi.getFilteredByGlob("src/content/posts/**/*.md")
        .filter(item => item.data.category === "教程");
    });
  }
};
```

### 集合排序
```js
// 按日期降序
const sorted = collection.getFilteredByTag("post")
  .sort((a, b) => b.date - a.date);
```

## 6. 过滤器（Filters）

### 注册过滤器
```js
// eleventy/config/filters.js
module.exports = {
  registerDateFilters(eleventyConfig) {
    eleventyConfig.addFilter("dateFormat", (date, format) => {
      return luxon.DateTime.fromJSDate(date).toFormat(format);
    });
  },
  registerTitleFilters(eleventyConfig) {
    eleventyConfig.addFilter("titleCase", (str) => {
      return str.replace(/\b\w/g, c => c.toUpperCase());
    });
  }
};
```

### 在模板中使用
```njk
{{ date | dateFormat("yyyy-MM-dd") }}
{{ title | titleCase }}
{{ collections.post | reverse | first }}
```

### 常用内置过滤器
- `| safe` — 输出原始 HTML（不转义）
- `| escape` — HTML 转义
- `| url` — URL 编码
- `| log` — 输出到控制台（调试用）

## 7. 短代码（Shortcodes）

### 普通短代码
```js
eleventyConfig.addShortcode("image", (src, alt) => {
  return `<img src="${src}" alt="${alt}" loading="lazy">`;
});
```

```njk
{% image "/assets/photo.jpg", "描述文字" %}
```

### Paired 短代码
```js
eleventyConfig.addPairedShortcode("callout", (content, type = "info") => {
  return `<div class="callout callout-${type}">${content}</div>`;
});
```

```njk
{% callout "warning" %}
  注意事项内容
{% endcallout %}
```

## 8. 插件

### 常用官方插件
- **`@11ty/eleventy-plugin-syntaxhighlight`**：代码高亮（Prism.js）
- **`@11ty/eleventy-img`**：图片优化（自动 WebP/AVIF、响应式尺寸）
- **`@11ty/eleventy-fetch`**：外部数据缓存
- **`@11ty/eleventy-plugin-bundle`**：CSS/JS 打包

### 注册插件
```js
const syntaxHighlight = require("@11ty/eleventy-plugin-syntaxhighlight");
eleventyConfig.addPlugin(syntaxHighlight);
```

### 异步插件
```js
const { default: mermaidPlugin } = await import("@kevingimbel/eleventy-plugin-mermaid");
eleventyConfig.addPlugin(mermaidPlugin);
```

## 9. Markdown 配置

### 推荐插件
```js
const markdownIt = require("markdown-it");
const markdownItFootnote = require("markdown-it-footnote");
const markdownItGitHubAlerts = require("markdown-it-github-alerts");
const markdownItMark = require("markdown-it-mark");
const markdownItTaskLists = require("markdown-it-task-lists");

const mdLib = markdownIt({ html: true, breaks: true, linkify: true })
  .use(markdownItFootnote)
  .use(markdownItGitHubAlerts.default)
  .use(markdownItMark)
  .use(markdownItTaskLists, { enabled: true, label: true });

eleventyConfig.setLibrary("md", mdLib);
```

### markdown-it 选项说明
- `html: true` — 允许 Markdown 中嵌入 HTML
- `breaks: true` — 换行符转 `<br>`
- `linkify: true` — 自动识别 URL 为链接

## 11. 性能优化

### 开发阶段
- 使用 `--incremental` 标志进行增量构建
- `npx eleventy --serve --incremental`

### 构建优化
- 使用 `eleventyConfig.addGlobalData("env", process.env)` 传递环境变量
- 避免在模板中执行昂贵的同步操作
- 使用 `@11ty/eleventy-fetch` 缓存外部 API 调用

### 图片优化
```js
const Image = require("@11ty/eleventy-img");

eleventyConfig.addShortcode("optimizedImage", async (src, alt) => {
  const metadata = await Image(src, {
    widths: [400, 800, 1200],
    formats: ["webp", "jpeg"],
    outputDir: "_site/assets/img/"
  });
  return Image.generateHTML(metadata, { alt, loading: "lazy" });
});
```

## 15. 注意事项

- **不要**在 `_includes` 中放置需要独立输出的页面文件
- **不要**在 front matter 中使用 `permalink` 与目录数据文件中的 `permalink` 冲突
- **不要**在模板中执行同步的文件 I/O 操作
- **优先使用**目录数据文件（`.11tydata.js`）而非在每个文件中重复 front matter
- **优先使用** Nunjucks 的 `extends` + `block` 实现布局继承，而非 `layout` front matter（更灵活）
- **使用** `| safe` 过滤器输出已信任的 HTML，避免 XSS
- **注意** CommonJS vs ESM：`package.json` 中 `"type": "commonjs"` 时用 `require()`，`"type": "module"` 时用 `import`
